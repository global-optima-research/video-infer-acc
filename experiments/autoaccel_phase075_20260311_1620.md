## 稀疏注意力实现

### Frame-Local Sparse Attention
基于 Phase 0 发现（同帧注意力占 78.5%，block diagonal ratio = 9x），实现帧级窗口化注意力：
- **Window=0**: 每帧仅关注自身（同帧注意力），理论稀疏度 80%
- **Window=1**: 每帧关注 ±1 邻帧，理论稀疏度 48%

自定义 `SparseWanAttnProcessor` 替换默认处理器，在 self-attention 中按帧分割 Q/K/V，对每帧独立计算窗口化注意力。

### Bug 修复记录
初始版本 spatial_tokens 计算错误（使用 //8），导致条件 `query.shape[1] == T * S` 永远不匹配，稀疏注意力从未被激活。修复为 //16 后确认生效（SSIM 变化 + 速度提升可见）。

## 交互分析 (Sparse × Cache)

## 分析与结论

### 发现 1: Frame-Local Sparse 对 Wan 模型质量破坏严重

即使 Phase 0 分析显示 78.5% 的注意力集中在同帧，**丢弃跨帧注意力在所有测试配置下均导致不可接受的质量损失**：

- 全层 Window=0 (80% 稀疏): SSIM = 0.34，完全不可用
- 全层 Window=1 (48% 稀疏): SSIM = 0.35，同样不可用
- 仅后 5 层 Window=0: SSIM = 0.55，仍不可用
- 最佳配置 S5 (W=1 last-10): SSIM = 0.57，仍远低于 0.90 阈值

**原因分析**：
- 注意力质量 ≠ 注意力权重分布。78.5% 同帧 ≠ 可以丢弃 21.5% 跨帧
- 跨帧注意力虽然权重小，但承载关键的时序一致性信息
- **双重累积效应**：误差在层间 AND 步间均累积（30 层 × 50 步 = 1500 次 sparse 计算）
- 层敏感度差异极大：Layer 0 单层 SSIM=0.56，Layer 25 单层 SSIM=0.92
- Window=1 并不比 Window=0 好多少，说明问题不在窗口大小而在方法本身

### 发现 2: Sparse×Cache 有超加性交互

交互比率 1.04-1.07，是三种技术组合中首个非正交结果：
- quant × cache: 0.99（正交，Phase 0.5）
- sparse × cache: 1.04-1.07（超加性，本实验）

**解释**：FBC 通过跳过整步计算来加速；当 FBC 不跳步时，稀疏注意力减少每步的计算量。FBC 跳步时，稀疏注意力的「节省」被放大（因为剩余需要实际计算的步数更少，每步节省的比例更大）。

### 发现 3: 加速 vs 稀疏层数近似线性

```
稀疏层数  Window=0 加速   Window=1 加速
5/30      1.03x          —
10/30     1.07x          1.04x
15/30     1.11x          1.06x
30/30     1.23x          1.12x
```

每增加 1 层稀疏 (Window=0) 约贡献 0.0077x 加速（线性拟合 R²≈0.99）。

### 发现 4: Quality-Speed Pareto 前沿极差

没有任何配置能同时满足 SSIM>0.90 和 speedup>1.05：

```
         SSIM=0.90 目标线
              ↓
  1.0  ┤ C0 ●
       │         (无法到达的理想区域)
  0.8  ┤
       │
  0.6  ┤    S5●  S3●
       │
  0.4  ┤   S4● S2●    C2● C1●
       │
  0.2  ┤  S1●
       └──┼──┼──┼──┼──┼──┼──→ 加速比
         1.0 1.05 1.10 1.15 1.20 1.25
```

### Go/No-Go 判定

| 维度 | 判定 | 说明 |
|------|------|------|
| Frame-local sparse 可用性 | ❌ 不可用 | SSIM 最高 0.57，远低于 0.90 |
| Sparse×Cache 交互 | ✅ 非正交 | 比率 1.04-1.07，搜索有价值 |
| 加速效果 | ⚠️ 微弱 | Window=0 全层仅 1.23x |
| Per-layer 差异化必要性 | ✅ 已证明 | 全层 vs 选择性差异显著 |
| 固定窗口策略 | ❌ 不足 | 需要更智能的稀疏策略 |

**综合判定：Frame-local 窗口化稀疏注意力 NO GO。需要更好的稀疏方法。**

## 下一步建议

1. **Dynamic Sparse Attention**: 基于实际注意力分数动态选择 top-k 帧（而非固定窗口），保留高权重的跨帧连接
2. **Step-adaptive Sparse**: 不同去噪步数使用不同稀疏策略（早期步需要更多全局信息，后期步可更稀疏）
3. **Attention Distillation**: 用全注意力的 pattern 作为 teacher，训练一个轻量级的 mask predictor
4. **换技术路线**: 如果 training-free sparse 都不行，考虑 token merging/pruning 或其他降低 attention 复杂度的方法
5. **AutoAccel 核心贡献重新定位**: sparse×cache 超加性已证明搜索价值；但需要至少一种 SSIM>0.90 的稀疏方法才能使组合有意义
