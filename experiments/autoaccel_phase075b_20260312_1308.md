## 分析与结论

### 发现 1: Step-Adaptive 无法挽救 Frame-Local Sparse

SSIM 随稀疏步数近似线性下降：

```
稀疏步数   SSIM     加速    每步 SSIM 损失
 0/50     1.000    1.00x   —
 5/50     0.830    1.01x   0.034/步
10/50     0.691    1.03x   0.031/步
15/50     0.602    1.05x   0.027/步
20/50     0.554    1.07x   0.022/步
25/50     0.492    1.10x   0.020/步
```

**即使只在最后 5 步应用 sparse，SSIM 也只有 0.83，且加速仅 1.01x（无实用价值）。**

要达到 SSIM=0.90 估计只能在最后 2-3 步用 sparse，加速 < 1.005x，完全无意义。

### 发现 2: Step-Adaptive × Layer-Selective 无改善

T6（层 10-29 + 步 25-49）SSIM=0.47 甚至比 T1（全层 + 步 25-49）SSIM=0.49 更差。两个维度的限制叠加并未带来改善。

### 发现 3: Step-Adaptive + FBC 速度交互仍存在

```
T1 sparse-only:   1.10x
FBC 0.03:         1.26x  (from Phase 0.5)
期望独立:          1.39x
T8 实际:          1.43x
交互比率:          1.03
```

超加性交互（1.03）在 step-adaptive 模式下仍然存在，但因 sparse 本身质量不可用，这一发现没有实用价值。

### 综合判定：Frame-Local Sparse Attention 全面 NO GO

三轮实验覆盖了所有可能的"救活"方案：

| 策略 | 最佳 SSIM（SSIM>0.90 约束下的加速） | 判定 |
|------|-------------------------------------|------|
| 全层全步 (Phase 0.75 Run A) | 0.34 (无法达到 0.90) | ❌ |
| Layer-adaptive (Run B) | 0.57 (无法达到 0.90) | ❌ |
| Step-adaptive (本实验) | 0.83 @ 1.01x (不实用) | ❌ |
| Step + Layer 组合 | 0.60 (无法达到 0.90) | ❌ |

**根本原因**：Frame-local 窗口化注意力丢弃了跨帧 token 的 K/V，而这些跨帧连接承载的时序一致性信息在每个层和每个去噪步都是必需的。78.5% 注意力权重在同帧 ≠ 可以丢弃 21.5% 跨帧。注意力权重的分布不等于信息的重要性分布。

### 对 AutoAccel 的影响

| 维度 | 状态 |
|------|------|
| FBC (cache) | ✅ 唯一可用技术，1.26x @ SSIM=0.90 |
| SageAttention2 (quant) | ❌ RTX 5090 上无效 |
| Frame-local sparse | ❌ 全面 NO GO（三轮实验确认） |
| Sparse × Cache 交互 | ✅ 已证明超加性，但 sparse 不可用 |
| AutoAccel 技术空间 | ❌ 仅一种可用技术，不足以支撑组合搜索 |

## 下一步建议

1. **PAB (Pyramid Attention Broadcast)**：不丢弃跨帧 token，而是缓存并复用注意力输出。与 FBC（缓存 block 输出）在不同粒度工作，可能有非平凡交互
2. **迁移到 H800**：SageAttention2 在 A100/H100 上有 2-4x 加速，可能恢复 quant 这条线
3. **重新评估方向**：如果 training-free 技术空间在当前硬件上太小，考虑转向其他研究方向
