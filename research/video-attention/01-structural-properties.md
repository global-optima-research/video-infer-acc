# 视频 DiT 注意力的 7 个结构性质

> 每个性质附带量化证据和来源

## 性质 ①：块对角线主导（空间局部性）

**现象**：注意力矩阵呈瓦片式重复模式，每个瓦片对应两个 latent frame 之间的注意力。对角块（同帧内注意力）远强于非对角块（跨帧注意力）。

**量化证据**：
- 对角块平均权重是非对角块的 **2.80x**（Efficient-vDiT, 在 Mochi/HunyuanVideo/CogVideoX/Wan 上验证）
- 跨不同 prompt，注意力分数索引有 **>90% 重叠**，说明这是**架构固有属性**而非输入依赖特征

**直觉解释**：同一帧内的 token 共享相同的视觉上下文（颜色、纹理、物体），自然产生更强的注意力关联。

**来源**：[Efficient-vDiT (arXiv:2502.06155)](https://arxiv.org/abs/2502.06155)

---

## 性质 ②：时间局部性衰减

**现象**：跨帧注意力随帧距离单调递减。离当前帧越远，注意力权重越低。

**量化证据**：
- 局部窗口 (12, 24, 24) 仅覆盖 **15.52%** 的 token 空间，但捕获 **70%** 的总注意力分数（STA, 在 HunyuanVideo 上测量）
- 帧 1→帧 2 的注意力 vs 帧 1→帧 10 的注意力差异随距离单调增大

**数学模型**：
- Radial Attention 证明注意力能量随时空距离**指数衰减**，拟合 R² > 0.985
- 这意味着理论上只有 **O(n log n)** 个有效注意力项（vs O(n²) 的全注意力）

**来源**：[STA (arXiv:2502.04507)](https://arxiv.org/abs/2502.04507), [Radial Attention (arXiv:2506.19852)](https://arxiv.org/abs/2506.19852)

---

## 性质 ③：跨 Prompt 模式不变性

**现象**：给定同一模型，不同文本 prompt 产生的注意力模式高度相似。模式是模型架构的固有属性，不是输入决定的。

**量化证据**：
- 跨不同 prompt 的注意力分数索引重叠 **>90%**（Efficient-vDiT）
- Head 特化模式（空间头/时间头）跨 prompt 一致（STA）
- Compact Attention 报告跨 prompt 模式相似度 **>0.8**

**启示**：可以**离线 profile 一次**，对所有输入复用。不需要每次推理都动态决定稀疏模式。

**来源**：[Efficient-vDiT](https://arxiv.org/abs/2502.06155), [STA](https://arxiv.org/abs/2502.04507), [Compact Attention (arXiv:2508.12969)](https://arxiv.org/abs/2508.12969)

---

## 性质 ④：去噪步间稳定性（U 形曲线）

**现象**：注意力模式在去噪过程中呈 U 形变化——开头和结尾变化大，中间 70% 几乎不变。

**量化证据**：
- **前 ~15% 步**：高变化（噪声初始化阶段）
- **中间 ~70% 步**：极低变化，注意力输出高度稳定
- **末尾 ~15% 步**：显著变化（细节精修阶段）

**注意力类型层级**：
- 空间注意力：变化**最大**（高频：边缘、纹理）
- 时间注意力：**中等**变化（运动、动态）
- 跨模态注意力：**最稳定**（文本-视频语义对齐）

**当前利用状况**：
- PAB 利用了这个性质，但仅适用于**分解式注意力**（spatial + temporal 分开的老架构）
- 对于 full 3D 注意力（Wan, HunyuanVideo 等主流模型），**没有方法**直接利用步间稳定性做注意力矩阵级的增量计算

**来源**：[PAB (arXiv:2408.12588)](https://arxiv.org/abs/2408.12588), [LiteAttention (arXiv:2511.11062)](https://arxiv.org/abs/2511.11062)

---

## 性质 ⑤：Head 功能特化

**现象**：不同注意力头承担不同功能角色，且这种特化跨 prompt 稳定。

**分类**：
| 头类型 | 注意力模式 | 功能 |
|--------|-----------|------|
| 空间头 | 帧内块状注意力 | 纹理/边缘/局部细节 |
| 时间头 | 跨帧等间距条纹 | 运动/时间一致性 |
| 全局头 | 近均匀分布 | 全局上下文 |
| Sink 头 | 集中在首个 text token | 冗余，可跳过 |

**量化证据**：
- Mochi 46-47 层 **>80% 的头** 是 sink 头（Analysis of Attention in VDiTs）
- 3-6% 的头可以完全跳过而不影响质量（Sparse-vDiT）
- 头类型在去噪过程中可能**动态切换**（SVG 发现 spatial/temporal 角色不固定）

**当前利用状况**：
- SVG 做了空间/时间二分类 → 太粗糙
- Compact Attention 分了 local/cross-shaped/global → 更细但没结合量化/分块
- 没有方法对不同类型的头使用**完全不同的计算策略**（如空间头用小窗口、时间头用 stride、sink 头直接跳过）

**来源**：[SVG (arXiv:2502.01776)](https://arxiv.org/abs/2502.01776), [arXiv:2504.10317](https://arxiv.org/abs/2504.10317), [Compact Attention](https://arxiv.org/abs/2508.12969)

---

## 性质 ⑥：高秩但结构化稀疏

**现象**：视频注意力矩阵是高秩的（不像语言模型那样低秩），但具有结构化稀疏性。

**量化证据**：
- 与语言模型不同，视频注意力的**多个奇异值都是显著的**（VMonarch）
- 但稀疏性是**结构化的**：可以用 Monarch 矩阵分解（块对角 × 置换的乘积）捕获
- Monarch 分解设 m=T（帧数）、b=H×W（空间 token），自然对齐视频结构

**层敏感性异质性**：
- 大多数层可容忍 **20%** 稀疏度而不降质
- **关键层（如 Mochi 44-45 层）**在仅 20% 稀疏度时就出现**像素化**
- 排除关键层后，其余层可到 **70% 稀疏度**

**警告**：视觉上看起来稀疏的层，功能上不一定可稀疏。小权重在关键层可能携带不成比例的信息。

**来源**：[VMonarch (arXiv:2601.22275)](https://arxiv.org/abs/2601.22275), [arXiv:2504.10317](https://arxiv.org/abs/2504.10317)

---

## 性质 ⑦：注意力熵分布

**现象**：注意力熵在不同头/层/模型之间差异巨大。

**量化证据**：
- **低熵**（近单位矩阵）→ 捕获结构属性，保持局部信息
- **高熵**（近均匀分布）→ 与更高视频质量相关
- AnimateDiff：>70% 时间注意力头熵接近 100%（近均匀）
- CogVideoX：熵范围 15-45%（更接近 identity）

**启示**：高熵头（近均匀）可以用常数近似而不损失太多信息；低熵头需要精确计算。

**来源**：[arXiv:2504.12027](https://arxiv.org/abs/2504.12027)

---

## 性质间的关联

```
① 块对角主导 ←→ ② 时间衰减       （同一现象的两个视角）
③ 模式不变性 → ⑤ 头特化是固有的   （可预编译）
④ 步间稳定性 → 增量计算的理论基础
⑤ 头特化    → 异构调度的依据
⑥ 高秩结构化 → 不能用简单低秩近似，需要 Monarch 或自适应分块
⑦ 熵分布    → 精度分配的依据（高熵头用低精度）
```

## 对统一框架的要求

一个统一框架需要同时建模：
1. **空间结构**（①②⑦）→ 决定分块策略和精度
2. **时间结构**（④）→ 决定跨步复用策略
3. **头结构**（⑤⑥）→ 决定每头计算策略
4. **模型结构**（③⑥）→ 决定哪些可以预编译
