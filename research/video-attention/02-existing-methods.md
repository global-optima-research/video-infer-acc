## 详细分析

### 1. STA (Sliding Tile Attention) — 当前最强 kernel

**核心思路**：把 token 组织成 3D 立方体 tile，tile 大小 T 满足 B = T³（B 是 FA block size）。同一 tile 内的 query 共享相同的 key 集合（局部窗口内的 tile）。

**利用的性质**：
- ① 空间局部性 → tile 化利用了块对角结构
- ② 时间局部性 → 窗口限制利用了衰减
- ③ 跨 prompt 不变性 → 离线校准每头窗口大小

**关键创新**：tile 化产生的都是**全密或全空块**，没有"混合块"，避免了传统 SWA 在 3D 中的 15%+ 开销。

**没利用的**：
- ④ 步间稳定性：每步从头计算
- ⑤ 头特化：虽然校准了窗口大小，但所有头用相同的算法（只是窗口不同）
- ⑥ 层敏感性：没有区分关键层和普通层
- ⑦ 熵→精度映射：没有

**性能**：
- Training-free: 54-58% 稀疏度安全
- With finetuning: 91% 稀疏度，需 2000 合成视频 + 8h × 8 H100
- Kernel: 2.8-17x vs FA2, 1.6-10x vs FA3

---

### 2. FPSAttention — 当前最强 E2E

**核心思路**：统一 FP8 量化 + 结构化稀疏，以 3D tile 为粒度。去噪步感知策略适配噪声 schedule。原生 Hopper kernel。

**利用的性质**：
- ① ② 空间时间局部性 → tile 化稀疏
- 量化 → 但不是基于 ⑦ 熵的自适应量化

**没利用的**：
- ③ 不变性：没有预编译
- ④ 步间稳定性：虽然步感知，但没做增量计算
- ⑤ 头特化：没有异构调度

**性能**：
- Kernel: 7.09x 加速
- E2E: Wan 2.1-14B 720p **4.96x** 加速
- NeurIPS 2025 Spotlight

---

### 3. SVG (Sparse VideoGen) — 头分类先驱

**核心思路**：在线 profiling（采样 1% token）分类每个头为空间型或时间型，然后施加对应的稀疏模式。

**利用的性质**：
- ⑤ 头特化 → 但只有空间/时间二分类

**没利用的**：
- 二分类太粗：遗漏了 cross-shaped、global、sink 类型
- Profiling 有 3% 运行时开销
- 不利用 ①②③④⑥⑦

**性能**：E2E 2.28-2.33x（HunyuanVideo）

---

### 4. VMonarch — 理论最优雅

**核心思路**：用 Monarch 矩阵（块对角 × 置换的乘积）近似注意力矩阵。设 m=T, b=H×W 对齐视频结构。交替最小化找最优 Monarch 近似。

**利用的性质**：
- ⑥ 结构化稀疏 → Monarch 分解捕获块结构

**没利用的**：
- 需要微调（非即插即用）
- 不利用 ③ 不变性（不预编译）
- 不利用 ④ 步间稳定性

**性能**：17.5x 注意力 FLOPs 减少，5x+ 注意力加速，90% 稀疏度

---

### 5. Radial Attention — 理论最有潜力

**核心思路**：形式化注意力能量随距离的指数衰减，证明只有 O(n log n) 个有效项。

**利用的性质**：
- ② 时间衰减 → 数学建模
- ⑦ 能量分布 → 自适应精度

**意义**：给出了**理论上界**，证明视频注意力的内在复杂度远低于 O(n²)

**没利用的**：实现层面的所有其他性质

---

## Gap 分析：无人覆盖的组合

```
性质组合              → 对应的优化策略         → 现有覆盖

③ + ⑤ → 预编译头专用 kernel          → 无
④ + 全3D → 注意力矩阵级增量计算      → 无（PAB仅限分解式）
⑤ + ⑦ → 头类型决定计算精度          → 无
① + ④ + ⑤ → 步感知 + 头感知分块      → 无
③ + ④ + ⑤ → 离线 profile → 编译 → 跨步复用  → 无

所有 5 个维度的统一                    → 无
```

## 关键启示

1. **STA 是最重要的基线**：利用了最多性质（3/7），kernel 最快
2. **FPSAttention 是最实用的竞品**：E2E 最快，NeurIPS Spotlight
3. **维度 3（跨步增量）和维度 5（预编译）是完全空白**
4. **真正的增量来自组合**：单个性质的 marginal gain 在递减，组合利用才有突破空间
5. **即插即用是关键**：需要微调的方法（VMonarch）采用率低
